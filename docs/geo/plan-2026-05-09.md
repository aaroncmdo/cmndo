## 1. Können LLMs direkt Termine buchen?

**Aktuell: nein.** LLMs können den Self-Dispatch-Flow zitieren und auf `claimondo.de/gutachter-finden` verlinken — aber den Booking-Vorgang selbst nicht ausführen, weil Browser-Geolocation, Termin-Auswahl-UI und Signatur-Canvas nötig sind.

### Drei Stufen LLM-Buchung (von simpel zu mächtig)

**Stufe 1 — MCP-Server (Model Context Protocol)** ⭐ empfohlen
- Exposed Functions: `findGutachter(plz, schadentyp)`, `bookTermin(...)`, `getStatus(fallId)`
- Konsumenten: ChatGPT (Enterprise/Custom-GPT), Claude Desktop, Cursor
- Aufwand: **2–3 Tage**
- Hebel: größter — MCP wird zunehmend Standard für AI-Agents
- Weiterer Vorteil: lässt sich auch für Dispatch-internen ChatGPT-Workflow nutzen

**Stufe 2 — `.well-known/ai-plugin.json` + `openapi.yaml`**
- Klassische ChatGPT-Plugin-Discovery
- Funktioniert bei ChatGPT-with-Browsing, weniger bei Perplexity/Claude
- Aufwand: **1 Tag**
- Hebel: mittel — ChatGPT-only

**Stufe 3 — API + AI-friendly Markdown-Docs unter `/api/docs`**
- Niedrigste Hürde, nutzt vorhandene `erstelleGutachterFinderAnfrage` Server-Action
- AI-Agents können via Function-Calling nutzen wenn ihnen das Schema bekannt ist
- Aufwand: **Stunden**
- Hebel: klein — eher "passive" Discoverability

### Empfehlung
Stufe 1 (MCP) **nach** dem GEO-Tier-1-Live + PageSpeed-Pass. Wenn MCP nicht in dieser Iteration: Stufe 3 als Quick-Win für AEO-Citation-Optimierung.
