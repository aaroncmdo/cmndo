> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Schließt 3 still kaputte Phase-Lookups (admin/faelle hub, kanzlei/mandate, kanzlei/kanban) + 2 stale Kommentare in einem PR gegen `staging`. Eliminiert die MP-8b id-Assumption-Bug-Klasse (claims.id ≠ faelle.id) aus den verbleibenden Callern der claims-zentrischen `v_claim_phase`.

**Architecture:**
- **Fix 1** (admin/faelle hub): **Strategie A** — `v_claim_listing` liefert `main_phase`/`sub_phase` bereits mit (siehe `lib/claims/types.ts:49-50`); SELECT erweitern, separaten `v_claim_phase`-Read komplett entfernen. Eliminiert die Bug-Klasse strukturell (keine phase-by-id-Map mehr) + 1 DB-Round-Trip weniger.
- **Fix 2 + 3** (kanzlei mandate/kanban): wiederverwende `getClaimPhaseMap(claimIds: string[])` aus `@/lib/claims/claim-phase-map` (etabliertes Pattern, Service-Client, typed enum-Casts intern). Inline-`admin.from('v_claim_phase')`-Read + inline-`toClaimMainPhase`/`toClaimSubPhase`-Aufrufe weg.
- **Fix 4 + 5**: reine Kommentar-Korrekturen.

**Tech Stack:** Next.js 15 RSC · Supabase-js · TypeScript · keine neuen Deps · keine Migration · keine DB-Tool-Calls.

**Test-Strategie (explizit):**
Keine automatisierten Unit-Tests pro Code-Fix. Begründung:
1. RSC-Pages (`default async function`) unit-zu-testen erfordert Mock-Infrastruktur, die in diesem Repo nicht etabliert ist + Refactor (data-transformation in Helper extrahieren), was den Spec-Scope erweitert.
2. **Strategie A für Fix 1** eliminiert die Bug-Klasse **strukturell** — kein phaseMap mehr, kein lookup-by-id → kann nicht mehr falsch keyed werden.
3. **Fix 2 + 3** nutzen `getClaimPhaseMap`, dessen File-Header die Invariante (`claims.id != faelle.id`) verbindlich dokumentiert — neue Caller folgen dem Vertrag.
4. **Task 6 — Audit-Sweep** ist das Bug-Class-Safety-Net: grep nach `.in('claim_id', ...)`-Pattern + Source-Variable-Inspektion. Findet ad-hoc Inline-Reads, die den Helper umgehen.
